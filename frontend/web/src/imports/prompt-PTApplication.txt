Update the product design and information architecture for this system with the following business requirements.

IMPORTANT PRODUCT CONTEXT
This is a web-based health-tech and fitness coaching platform called “Gymnini – AI Gym Coach”.
The product supports Clients, PT users, and Admin.
PT is not a separate isolated identity. PT is a Client user with additional trainer capabilities.
The UI should be sporty, modern, premium, dark-mode first, with a black / charcoal + green fitness-tech visual style.

==================================================
1. INBODY MODULE – TWO INPUT METHODS
==================================================

The InBody feature must support TWO different input methods:

METHOD A – MANUAL ENTRY
The user can manually enter InBody/body composition data through a form.

METHOD B – IMAGE UPLOAD
The user can upload a photo of a real InBody paper result.
The system will extract data from the image, then the user reviews and confirms the extracted values before saving.

Design the InBody experience to clearly support both methods.

REQUIRED INBODY UX
Design these screens/states:

1. InBody landing / overview page
- show latest record
- show history
- show body metric trends
- show CTA options:
  - “Enter Data Manually”
  - “Upload InBody Image”

2. Manual Entry flow
- clear data-entry form
- sectioned layout
- easy-to-edit fields
- save draft / submit / cancel states
- validation errors
- confirmation state

3. Image Upload flow
- upload image area
- drag & drop or browse file
- image preview
- OCR / extraction processing state
- extraction review screen
- manual correction form after extraction
- confidence / status indicators
- confirm and save state
- failed extraction fallback to manual correction

4. InBody data display
- latest metrics cards
- comparison between two records
- trend charts over time
- segmental analysis visualization
- clear highlighting of progress / regression

Important extracted or entered fields should include:
- weight
- height
- skeletal muscle mass
- body fat mass
- segmental lean analysis
- segmental fat analysis

REQUIRED INBODY STATUS STATES
Show clear visual states for:
- no records yet
- manual entry draft
- image uploaded
- processing
- extracted
- needs confirmation
- saved successfully
- extraction failed
- manually corrected

The UX must make it obvious that the user can choose either:
- manual entry
- image upload

==================================================
2. PT APPLICATION / BECOME A PT FLOW
==================================================

The “Become a PT” flow must be redesigned to collect much more information.

IMPORTANT BUSINESS LOGIC
A normal Client can apply to become a PT.
This is not an instant upgrade anymore.
It is an application / approval workflow.

The system must collect detailed PT application data.
Admin will review and approve/reject this application.

Design a full PT application flow with rich profile and verification data.

REQUIRED PT APPLICATION FIELDS
The PT application form must include:

A. BASIC PERSONAL / VERIFICATION INFORMATION
- phone number
- national ID / citizen ID number
- portrait photo / profile headshot
- current address

B. PROFESSIONAL BACKGROUND
- years of experience as a PT
- main specialties
  Examples:
  - muscle gain
  - fat loss
  - powerlifting
  - bodybuilding
  - calisthenics
  - yoga
  - rehab
  - sports nutrition
- relevant education background
- professional certifications
- certification issuing organization
- whether certification is still valid / expiration status
- portfolio / capability profile / LinkedIn / personal website

C. SUPPORTING FILES
- certificate image(s) or PDF file(s)
- optional identity verification images if needed
- support for future citizen ID scan / identity verification

D. TARGET CLIENT GROUPS
- who the PT wants to coach
  Examples:
  - beginners
  - weight-loss clients
  - postpartum women
  - advanced trainees
  - office workers
  - athletes
  - rehab clients

E. SERVICE / WORKING INFORMATION
- available schedule / available coaching time slots
- working area
- online / offline / hybrid
- preferred service area / location
- desired price / session fee

F. PROFESSIONAL BRANDING
- professional social media accounts if available
- Instagram
- Facebook page
- TikTok
- YouTube
- LinkedIn
- personal fitness brand links

APPLICATION STATUS STATES

5. CLIENT / PT / ADMIN ROLE LOGIC
==================================================

Keep the existing correct business logic:

- Client is the default user
- PT is a Client with additional PT permissions
- PT still keeps normal Client features
- PT should be able to switch between:
  - Client View
  - PT View
- PT should not see “Find PT / Hire PT”
- Admin has management and approval access

Update the interface so the new PT application / PT approval flow fits naturally into this role system.
